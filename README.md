# Siril Batch Stacking Pipeline

> **Note:** The code and structure in this project were generated by AI (Gemini 3.1 Pro Preview in Google AI Studio).

This project is an automated astrophotography stacking pipeline built using Python and the `pysiril` library. It interfaces directly with the Siril desktop application backend to process massive amounts of light frames without running out of disk space. 

By breaking the processing down into manageable batches, it automatically converts, calibrates, registers, and partially stacks groups of frames, deleting intermediate bloated files along the way. Finally, it aligns and stacks the partial batches into a single master image.

## Features
* **Memory & Storage Efficient:** Processes light frames in smaller batches (configurable) to prevent massive disk-space consumption from `.seq` and `.fit` intermediate files.
* **Format Agnostic:** Natively supports standard astrophotography formats like `.fit`, as well as Camera RAW files (like Olympus `.orf`, `.cr2`, `.nef`, etc.) without premature debayering.
* **Advanced Calibration:** Uses Master Dark and Master Flat frames with automatic CFA equalization and cosmetic correction (hot/cold pixel removal).
* **Two-Pass Global Registration:** Calculates alignment for maximum framing before physically exporting the transformed images.
* **Weighted Stacking:** Rejects poor frames based on FWHM, weighs partial stacks by FWHM, and weighs the final master stack by the total number of underlying exposures.
